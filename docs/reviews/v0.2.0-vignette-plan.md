## Subsection 4.1 — Generating Theoretical Reference SSDs

**Functions:** `generate_ssd_weibull()`, `generate_ssd_reverse_j()`, `generate_ssd_bimodal()`,
`stand_qmd()`, `stand_basal_area()`, `SG_DBH_BREAKS`, `SG_DBH_MIDPOINTS`, `SG_DBH_THETA`

**eval=TRUE** — pure R, no keras.

### Methodological context key points
- Parts 1–3 use real inventory stands; generators create *theoretical* archetypes from first
  principles with N (stems/ha) and BA (m²/ha) as the only inputs
- Cao-Baldwin constrained least-squares solver guarantees exact conservation of N and BA
- Three archetype types ecologically: Weibull = even-aged / unimodal; reverse-J = all-aged /
  selection forest; bimodal = two-cohort / mixed structure
- Use cases distinct from existing functions: (a) controlled benchmarks to test model performance
  on idealised structures; (b) checking model generalisation beyond training distribution;
  (c) finding closest theoretical archetype to a predicted SSD

### Code chunk outline (eval=TRUE)
```r
N  <- 400   # stems/ha
BA <- 28    # m²/ha

dG <- stand_qmd(N_target = N, BA_target = BA)

ssd_weibull   <- generate_ssd_weibull(c_shape = 2.5, N_target = N, BA_target = BA)
ssd_reverse_j <- generate_ssd_reverse_j(q = 1.3, N_target = N, BA_target = BA)
ssd_bimodal   <- generate_ssd_bimodal(r1 = 0.5, r2 = 1.5, N_target = N, BA_target = BA)

# Verify: both N and BA must match targets exactly
cat("Weibull   — N:", round(sum(ssd_weibull), 1),
    "  BA:", round(stand_basal_area(ssd_weibull), 2), "\n")
cat("Reverse-J — N:", round(sum(ssd_reverse_j), 1),
    "  BA:", round(stand_basal_area(ssd_reverse_j), 2), "\n")
cat("Bimodal   — N:", round(sum(ssd_bimodal), 1),
    "  BA:", round(stand_basal_area(ssd_bimodal), 2), "\n")
```
Then a `ggplot2` `geom_col` comparison of the three, using `SG_DBH_MIDPOINTS` as x-axis labels
(ggplot2 already loaded in Section 2 — no new imports).

## Subsection 4.2 — Classifying Forest Structure Archetypes

**Functions:** `classify_ssd()`

**eval=TRUE** — pure R, using carry-forward SSDs from 4.1.

⚠️ **V3 correction applied:** `classify_ssd(ssd_abs, stem_number)` requires ABSOLUTE stem counts
and a required `stem_number` argument (no default). Returns a **list** — access archetype via
`$category`. The generators from 4.1 provide the correct input format (absolute counts, N known).
Classifying test-data proportions directly is NOT possible without unscaled N — show that as
`eval=FALSE`.

### Methodological context key points
- `analyze_and_visualize_predictions` (Part 2) shows *how much* predictions deviate numerically;
  `classify_ssd` answers whether the predicted structure *type* is ecologically correct
- Four classes (Reverse-J, Unimodal, Bimodal, Irregular) represent distinct stand developmental
  states with different management implications — mislabelling an Irregular as Unimodal carries
  real ecological meaning
- In a real validation workflow, classifying both observed and predicted SSDs enables an
  archetype-level confusion table across the test set

### Code chunk outline (eval=TRUE) — classify generated reference SSDs

## Subsection 4.3 — Multi-Metric Prediction Evaluation

**Functions:** `compute_prediction_metrics()`, `jsd_divergence()`, `perturb_features()` (eval=FALSE)

**eval=TRUE** for `compute_prediction_metrics` / `jsd_divergence` — pre-computed predictions
in `top_model_predictions`; `stem_number.ha` (absolute stems/ha column confirmed present)
converts proportions to absolute counts.

**eval=FALSE** for `perturb_features` — requires scalers object absent from `deepReveal_example_data`.

### Methodological context key points
- `kl_divergence_metric()` (ADR-001): single scalar, used during training and PFI; asymmetric,
  unbounded — can blow up when predicted distribution assigns near-zero prob to an observed class
- `analyze_and_visualize_predictions`: visual per-stand inspection (Parts 1–2)
- `compute_prediction_metrics()`: programmatic 4-metric tibble (KL, JSD, RMSE, R²) for batch
  evaluation and tabular reporting across external validation sets
- JSD is symmetric and bounded in [0, ln 2] — directly comparable across stands with different
  N and structure; robust to zero predictions unlike KL
- Note: KL remains the primary training and ranking metric per ADR-001; JSD/RMSE/R² are complementary

⚠️ **V2 correction applied:** `stem_number.ha` is normalised (range 0.59–0.90), not absolute
stems/ha. No unscaled absolute stem density exists in `deepReveal_example_data`. The original
chunk multiplying proportions × `stem_number.ha` would produce N_total < 1 and silently return
all-NA. Fix: use generators (absolute counts, N exactly known) for the self-contained demo.

### Code chunk for `jsd_divergence` (eval=TRUE) — operates on proportions
```r
data("deepReveal_example_data")
preds       <- deepReveal_example_data$top_model_predictions
target_cols <- deepReveal_example_data$target_features
pred_cols   <- paste0(target_cols, "_pred")

# JSD between two stands' model predictions
p <- as.numeric(preds[1, pred_cols])
q <- as.numeric(preds[2, pred_cols])
cat("JSD (stand 1 vs stand 2):", round(jsd_divergence(p, q), 4),
    "nats  [0, ln(2) =", round(log(2), 4), "]\n")

# JSD between observed and predicted for the same stand
p_obs  <- as.numeric(preds[1, target_cols])
p_pred <- as.numeric(preds[1, pred_cols])
cat("JSD (observed vs predicted, stand 1):",
    round(jsd_divergence(p_obs, p_pred), 4), "nats\n")
```

### Code chunk for `compute_prediction_metrics` (eval=TRUE) — generator-based demo
```r
# compute_prediction_metrics requires absolute stem counts (not proportions).
# The generators produce SSDs with exactly known N and BA — ideal for a demo.
N        <- 400
true_abs <- generate_ssd_weibull(c_shape = 2.5, N_target = N, BA_target = 28)

# Simulate a "predicted" distribution with a different Weibull shape
pred_ssd <- generate_ssd_weibull(c_shape = 3.0, N_target = N, BA_target = 28)
pred_rel <- pred_ssd / sum(pred_ssd)   # normalise to proportions (as a model would output)

metrics <- compute_prediction_metrics(true_abs = true_abs,
                                      pred_rel = pred_rel,
                                      N_total  = N)
print(metrics)
# In a real workflow: true_abs comes from field-measured class counts;
# pred_rel from your model's softmax output; N_total = sum(true_abs)
```
